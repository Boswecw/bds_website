# BDS Homepage Wireframe — HUD Integrated Design v1

Status: Conceptual Wireframe (UX-Theory Grounded)
Tone: Authoritative • Structured • Calm • Intelligent
Theme: Dark Navy Dominant
Assistant: Polite • Feminine • Capable • Non-intrusive

---

# 1. Core Design Philosophy

This homepage is not a marketing splash page.
It is a structured authority interface.

The HUD assistant is:
- Ambient by default
- Context-aware
- Non-intrusive
- Never sales-driven

Primary UX Principles Applied:
- Hick's Law (minimal choice density)
- Fitts's Law (clear primary action sizing)
- Progressive Disclosure
- Strong visual hierarchy
- Controlled whitespace rhythm

---

# 2. Global Layout Grid

Max width: 1200px
12-column grid
Consistent 8px spacing scale
Large outer margins for authority and breathing room

Primary vertical rhythm: 96px section spacing

---

# 3. Top Navigation

Left:
- BDS Logo

Right:
- Products
- Store
- Security
- About
- Contact

Far Right Edge:
- HUD Presence Indicator (vertical edge dock)

No admin link.
No clutter.

---

# 4. Hero Section (Authority Block)

Layout: Two-column structure

Left Column:
Headline (large, controlled weight)
Subtext (max 2 lines)
Primary CTA (solid orange)
Secondary CTA (ghost style)

Right Column:
Minimal geometric accent or subtle system visualization

Large whitespace below.

No animation chaos.

---

# 5. HUD Placement & Behavior

Default State:
- Slim vertical dock on right edge
- Small circular assistant indicator
- Soft pulse every 12–15 seconds (very subtle)

Expanded State:
- Slides in from right
- Max width: 32% viewport
- Slight background translucency
- Clean structured layout

HUD Content Structure:
- Title: "BDS Assistant"
- Short composed greeting
- 3 contextual suggestion buttons
- Input field below

Tone Example:
"How may I assist you today?"

No emojis.
No casual tone.

---

# 6. Products Section

3-column grid

Each card:
- Product title
- 1-line description
- Subtle border
- Minimal hover elevation

On hover:
HUD subtle suggestion (non-intrusive):
"Would you like a brief overview of ForgeCommand?"

---

# 7. Security & Infrastructure Section

Background: Slightly lighter navy surface

Left Column:
Structured bullet list:
- Passkey Authentication (FIDO2)
- Stripe-hosted payments
- Ed25519 signed admin control
- Private infrastructure segmentation

Right Column:
Minimal line-based architecture diagram

CTA: "Explain our security model"
Triggers HUD contextual explanation.

---

# 8. Authority Section

Professional background summary
Concise credentials
Structured layout

HUD suggestion:
"Would you like a summary of our governance philosophy?"

---

# 9. Footer

Structured columns:
- Legal
- Privacy
- Security
- Contact

Minimal.
No decorative clutter.

---

# 10. Motion Guidelines

Micro-interactions only:
- Button hover (150–200ms)
- Subtle card lift
- Controlled panel slide

Easing:
cubic-bezier(0.4, 0, 0.2, 1)

No scroll-trigger theatrics.

---

# 11. Visual Guardrails

Avoid:
- Neon tech aesthetic
- Gradient noise backgrounds
- Excessive glow
- Floating chat bubble style
- Sales-heavy language

Emphasize:
- Clean grid discipline
- Controlled accent usage
- Precision spacing
- Calm authority

---
